Hi, and thank you for the review. All three points have been addressed. The
security items (nonces, escaping) were resolved in the 2.0.2 release, and 2.1.3
makes the source-code reference more prominent. Details below, with file and line
references against the current submission.

## 1. Publicly documented resource for compiled content

The uncompressed source ships in the plugin and is documented in the readme:

- `src/index.js` and `webpack.config.js` are included in the plugin ZIP
  (`.distignore` explicitly keeps `src/` in the build).
- `readme.txt` has a `== Source Code ==` section — now moved directly below the
  feature list (near the top) so it is easy to find — linking to the public repo:
  https://github.com/imagewize/warder-cookie-consent
- As of 2.1.3, the only compiled asset, `dist/cookieconsent.bundle.js`, now begins
  with a comment banner pointing back to the source and repository, so the source
  location is visible from inside the compiled file itself:

  ```
  /*!
   * Warder Cookie Consent v2.1.3
   * This file is compiled from human-readable source with webpack.
   * Source: src/index.js (shipped in this plugin) + webpack.config.js
   * Repository: https://github.com/imagewize/warder-cookie-consent
   * Build from source: npm install && npx webpack
   */
  ```

  This banner is generated by webpack's `BannerPlugin` (see `webpack.config.js`),
  so it regenerates on every build and is never hand-edited.

## 2. Nonces and user permissions

Every action handled by `warder_handle_admin_actions()` in `inc/ajax.php` verifies
a nonce *before* any state is changed, paired with `current_user_can('manage_options')`:

| Action          | Nonce verification (`inc/ajax.php`)                                   | Nonce emitted (`inc/admin.php`)        |
|-----------------|----------------------------------------------------------------------|----------------------------------------|
| Add category    | `check_admin_referer('warder_add_category', 'warder_category_nonce')` (line 46) | `wp_nonce_field(...)` (line 413)       |
| Add cookie      | `check_admin_referer('warder_add_cookie', 'warder_cookie_nonce')` (line 64)     | `wp_nonce_field(...)` (line 359)       |
| Delete category | `wp_verify_nonce($nonce, 'delete_category_' . $category_id)` (line 85)           | `wp_nonce_url(...)` (line 219)         |
| Delete cookie   | `wp_verify_nonce($nonce, 'delete_cookie_' . $category_id . '_' . $cookie_index)` (line 102) | `wp_nonce_url(...)` (line 306) |

The AJAX settings save (`warder_ajax_save_settings`) also calls
`check_ajax_referer('warder_options_group-options', '_wpnonce')` (line 14) and
re-checks `current_user_can('manage_options')`. Failed delete-nonce checks call
`wp_die()` so the request cannot proceed. The permission check runs inside the
handler (not on every page load).

## 3. Escaping echoed output

The cited function, `warder_get_preferences_toggle_css()` in `inc/frontend.php`
(line 57), now returns **static, hardcoded CSS with no user input**. The toggle
position is applied via fixed CSS classes (`.warder-preferences-toggle--bottom-right`,
etc.), and the class name is escaped with `esc_attr()` where the button is printed
(`inc/frontend.php`, line 104). There is therefore no dynamic data flowing into the
CSS string.

Across the admin and frontend output, values are escaped at output time using the
context-appropriate helpers — `esc_html()` / `esc_html_e()`, `esc_attr()` /
`esc_attr_e()`, `esc_url()`, `esc_textarea()`, and `esc_js()`.
